# SDK identification harness

Identifies NitroSDK functions in the KH Days ROM by compiling reference SDK
source (from [`pret/pokediamond`](https://github.com/pret/pokediamond)) with
our matching `mwccarm 3.0_patch4` toolchain and searching the produced bytes
in `build/func_index.json` (generated by `tools/index_funcs.py`).

Result (979 unique `func_XXXXXXXX` named, ~4.2% of the 23,240 ROM functions):
`build/names.json` (gitignored — regenerate with `python sdk/build_all.py`).

## Workflow

1. Clone reference SDK source (one-time, not committed):
   ```sh
   git clone --depth 1 --filter=blob:none --sparse https://github.com/pret/pokediamond scratch/pokediamond
   cd scratch/pokediamond && git sparse-checkout set arm9/lib include
   ```
2. Stage the SDK module(s) of interest into `sdk/src/`:
   ```sh
   cp scratch/pokediamond/arm9/lib/NitroSDK/src/MI_*.c sdk/src/
   cp scratch/pokediamond/arm9/lib/NitroSDK/include/*.h sdk/include/
   cp scratch/pokediamond/include/nitro/*.h sdk/include/nitro/
   ```
3. Compile + byte-match everything in `sdk/src/`:
   ```sh
   python sdk/build_all.py
   ```
   Writes `sdk/build/identified.json` (per-file detail) and `sdk/build/names.json`
   (`func_X -> SDK name` mapping).

## How matching works

`mwccarm` emits each function in its own `.text` section. `build_all.py` iterates
every `STT_FUNC` symbol by `shndx` and:
- **reloc-free funcs**: exact byte match against `func_index.json` hex.
- **funcs with calls**: mask 4 bytes at each reloc offset, require the index
  entry has the same reloc-offset shape (callee names not enforced — chained
  identification can verify later).

Each match is byte-exact; dedup expansion is automatic because identical bytes
in the ROM (e.g. SDK code copied across overlays) all surface in one search.

## Stubs (committed, written here)

- `include/code32.h` / `code16.h` — `#pragma thumb off`/`on`.
- `include/function_target.h` — empty (some pret files reference it).
- `include/main.h`, `include/mmap.h` — empty/forwarder stubs.
- `nitro/types.h` (third-party) has been augmented locally with `ALIGN(n)` and
  `ENUMS_ALWAYS_INT_{ON,OFF,RESET}` macros (the originals expect these to come
  from the project's build env).

and (b) the 3D-geometry coprocessor path. Those are agent-territory.
